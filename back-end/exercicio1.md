exercícios da lista 1

1. O que é o Node.js?

é um ambiente de tempo de execução (runtime) de código aberto e multiplataforma, baseado no motor V8 do Google Chrome, que permite executar JavaScript no lado do servidor (backend).

2. O Node.js usa qual linguagem de programação?

O Node.js utiliza a linguagem de programação JavaScript. Ele é um ambiente de execução (runtime) que permite rodar código JavaScript no servidor (backend) e fora do navegador, utilizando o motor V8 do Google Chrome.

3. O Node.js roda no navegador ou fora dele?

O Node.js roda fora do navegador (no servidor ou diretamente na máquina/sistema operacional)

4. Para que serve o Node.js?

O Node.js serve principalmente para criar aplicações de rede escaláveis e de alta performance, permitindo executar JavaScript no lado do servidor (back-end), fora do navegador. Ele é ideal para aplicações em tempo real (chats, streaming), APIs REST e microsserviços, utilizando um modelo assíncrono e não bloqueante

5. Qual comando é usado para executar um arquivo Node.js no terminal?

O comando usado para executar um arquivo Node.js no terminal é node seguido do nome do arquivo

6. O que é o npm?

O npm (Node Package Manager) é o gerenciador de pacotes padrão para o ambiente de execução Node.js, instalado automaticamente com ele. Ele funciona como uma vasta biblioteca online (registro) de projetos JavaScript de código aberto e uma ferramenta de linha de comando (CLI) para instalar, compartilhar e gerenciar dependências, versões e scripts de projetos, facilitando o desenvolvimento front-end e back-end. 

7. O que significa a palavra módulo no Node.js?

No Node.js, um módulo é um arquivo JavaScript, ou um conjunto de arquivos/pastas, que encapsula uma funcionalidade específica e reutilizável, permitindo a organização, modularização e manutenção do código. Em vez de manter todo o código em um único arquivo gigante, o Node.js permite dividir o programa em unidades funcionais menores e organizadas.

8. O Node.js é mais usado no front-end ou no back-end?

O Node.js é primordialmente utilizado no back-end (servidor), sendo um ambiente de execução popular para criar aplicações web rápidas, escaláveis e em tempo real, utilizando JavaScript. No entanto, ele é fundamental no front-end moderno para rodar ferramentas de desenvolvimento (como Webpack, Vite) e frameworks Next.js

9. O que é o arquivo package.json?

O arquivo package.json é um componente fundamental no ecossistema Node.js e no desenvolvimento JavaScript, funcionando como o "coração" ou a "certidão de nascimento" de um projeto. Ele é um arquivo de texto no formato JSON (JavaScript Object Notation) localizado na raiz do projeto e contém metadados relevantes, além de listar as dependências que o projeto utiliza.

10. O Node.js é gratuito ou pago?

O Node.js é totalmente gratuito e de código aberto (open-source)
